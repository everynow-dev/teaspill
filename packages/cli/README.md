# @teaspill/cli

The `teaspill` binary (T6.2) — the dev loop plus entity inspection. A **thin
consumer** of the SDKs: it reuses `@teaspill/agents-sdk`
(`serve`/`registerDeployment`) and `@teaspill/frontend-sdk`
(`createActionsClient`, `createAgentCatalog`, `createAgentTimeline`) and never
reimplements stream reading, the actions client, or catalog shapes.

## Commands

| Command                               | What it does                                                                                                                                                |
| ------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `teaspill dev` (alias `platform-dev`) | `docker compose up -d` → **wait on gateway health** → **register local deployment(s) with retry + backoff** → tail logs. `--watch` re-registers on rebuild. |
| `teaspill agents ls`                  | List entities from the catalog (Electric shapes via the gateway `/shapes/*`). Filters: `--type`, `--status`, `--parent`, `--json`.                          |
| `teaspill spawn <type> [args]`        | Spawn an entity (`args` = JSON or a bare string). `--id` for idempotent spawn, `--parent`, `--idempotency-key`.                                             |
| `teaspill send <url> <message>`       | Message wake (`message` = JSON or string).                                                                                                                  |
| `teaspill control <url> <verb>`       | `interrupt` \| `pause` \| `resume` \| `archive` (`--reason`).                                                                                               |
| `teaspill logs <url>`                 | Follow + render an entity's timeline stream (messages, tool calls, run boundaries, errors) in seq order. `--deltas`, `--from-snapshot <seq>`.               |

`--help` (and per-command `--help`) is generated by [`cac`](https://github.com/cacjs/cac),
a tiny zero-dependency ESM parser. The parser lives at the edge only — every
command body is a plain function taking an injected `CliDeps`, so parse/dispatch
and the register backoff are unit-tested with fakes and no live stack.

## The `dev` sequencing — the register-before-up race

PLAN §5 T6.2 / D6: the electric-agents boot-order bug ("Stream not found" when a
deployment registers before the gateway/Restate is ready) is avoided by
**waiting on gateway health before registering** and **registering with
exponential backoff**. `agents-sdk`'s `registerDeployment` does a single attempt;
this CLI wraps it (`register.ts`: `waitForHealthy` + `retryWithBackoff`) so a
not-yet-ready gateway or not-yet-listening deployment retries instead of failing
the loop. Covered by `register.test.ts` (a fake probe that fails N times then
succeeds) and `cli.test.ts` (`runDev` end-to-end with fakes).

## Watch mode — the rebuild trigger contract

`teaspill dev --watch` observes the deployment's **built output directory**
(`--watch-path`, default `dist`). The CLI does **not** run the build; the
developer's own build/bundler writes fresh output there, and any file change
under the watched path — debounced (300 ms) — re-runs registration with backoff.
Restate picks up the fresh deployment revision on the next invocation. See
`compose.ts` `watchForRebuild`.

## Configuration

Client-side env (flags override env):

| Flag                       | Env                       | Default                            | Meaning                                                                                       |
| -------------------------- | ------------------------- | ---------------------------------- | --------------------------------------------------------------------------------------------- |
| `--gateway <url>`          | `TEASPILL_GATEWAY_URL`    | `http://localhost:8787`            | Gateway base URL (compose `GATEWAY_PORT=8787`).                                               |
| `--api-key <key>`          | `TEASPILL_API_KEY`        | —                                  | Gateway API key (`Authorization: Bearer`, D6). Required for every route except `GET /health`. |
| `--tenant <tenant>`        | `TEASPILL_TENANT`         | `default`                          | Deployment tenant (matches the gateway's `TEASPILL_TENANT`).                                  |
| `--deployment <url>` (dev) | `TEASPILL_DEPLOYMENT_URL` | `http://host.docker.internal:9080` | Deployment URL(s) to register (repeatable / comma-separated).                                 |

**host.docker.internal stance** (`work/plans/0001-build-v1/notes/self-hosting-networking.md` §3): Restate
dials registered URIs from _inside_ the container, so a `localhost` deployment
URL resolves to the container itself and every invocation fails. `dev` defaults
to `host.docker.internal` and **warns** (never rewrites — the gateway
deliberately does no rewriting) when a loopback URL is supplied.

## Dev-only / manual behaviors (not unit-tested)

`docker compose up -d` and `docker compose logs -f` shell out to Docker through
an injectable `Spawner` (`compose.ts`) — the SEQUENCING is tested with a fake
spawner, but no test drives a real daemon. Bringing the real stack up and
tailing real logs is exercised manually (or by the T6.3 conformance kit against
a live stack).
